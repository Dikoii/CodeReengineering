Dispensables

Elemen dalam kode yang tidak diperlukan atau redundan, yang dapat dihapus tanpa mengubah fungsionalitas atau kualitas dari kode tersebut. Contohnya adalah komentar yang menjelaskan kode yang sudah jelas, kode yang tidak pernah dipanggil atau digunakan, atau penggunaan variabel yang tidak perlu. Menghilangkan dispensables dapat membantu membersihkan dan meningkatkan kejelasan kode.

Comments

Komentar yang terlalu banyak atau tidak relevan dapat menjadi masalah karena mereka cenderung usang seiring waktu dan bisa jadi tidak akurat. Kode yang bersih seharusnya dapat menjelaskan dirinya sendiri tanpa terlalu banyak mengandalkan komentar.

Treatment

  • Extract Variable
  • Extract Method
  • Rename Method
  • Introduce Assertion

Contoh, terdapat komentar yang menjelaskan secara redundan apa yang sudah jelas dari kode. Solusinya, kita dapat menghapus komentar tersebut karena kode sudah cukup jelas tanpa perlu penjelasan tambahan.

Before

                // Order.java:
// This method calculates the sum of two numbers
public int add(int a, int b) {
    // Add two numbers and return the result
    return a + b;
}

After

                // Order.java:
public int add(int a, int b) {
    return a + b;
}

Duplicate Code

Ini terjadi ketika ada fragmen kode yang identik atau sangat mirip di berbagai bagian sistem. Hal ini bisa menjadi masalah karena mengubah satu bagian kode mungkin tidak mencerminkan perubahan di bagian duplikatnya, yang dapat menyebabkan ketidaksesuaian dan bug.

Treatment

  • Extract Method then Pull Up Field
  • Pull Up Constructor Body
  • Form Template Method
  • Substitute Algorithm
  • Extract Class
  • Extract Superclass
  • Consolidate Conditional Expression and use Extract Method
  • Consolidate Duplicate Conditional Fragments

Contoh, terdapat kode yang sama untuk menghitung luas lingkaran dan persegi panjang di berbagai tempat. Solusinya, kita dapat menggabungkan logika yang sama ke dalam satu method calculateArea untuk menghindari duplikasi kode.

Before

            // Order.java:
public double calculateCircleArea(double radius) {
    return Math.PI * radius * radius;
}

public double calculateRectangleArea(double length, double width) {
    return length * width;
}

After

            // Order.java:
public double calculateArea(String shape, double... dimensions) {
    if ("circle".equalsIgnoreCase(shape)) {
        return Math.PI * dimensions[0] * dimensions[0];
    } else if ("rectangle".equalsIgnoreCase(shape)) {
        return dimensions[0] * dimensions[1];
    }
    return 0; // Handle other shapes
}

Lazy Class

Kelas yang hanya memiliki sedikit atau tidak ada fungsionalitas yang digunakan juga dianggap sebagai "bad smell" karena dapat menambah kompleksitas tanpa memberikan nilai tambah yang signifikan.

Treatment

  • Inline Class
  • Collapse Hierarchy

Contoh, terdapat class `AddNumber` yang hanya memiliki satu method sederhana. Solusinya, kita memindahkan method tersebut ke class yang lebih relevan, seperti Calculator, untuk menghilangkan kelas yang tidak memberikan nilai tambah.

Before

            // Order.java:
class AddNumber {
    // Hanya mempunyai satu method
    public int add(int a, int b) {
        return a + b;
    }
}

After

            // Order.java:
// Move add method to a more relevant class
class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    // Memiliki method lain seperti perkalian, pengurangan, pembagian, dll.
}

Data Class

Kelas yang hanya berisi data tanpa adanya metode atau fungsionalitas tambahan sering kali dianggap sebagai masalah karena mereka tidak menawarkan perilaku atau logika yang dapat dimanfaatkan.

Treatment

  • Encapsulate Field
  • Encapsulate Collection
  • Move Method and Extract Method
  • Remove Setting Method and Hide Method

Contoh, terdapat class `Person` yang hanya berisi atribut dan setter getter tanpa logika tambahan. Solusinya, kita dapat menambahkan metode baru `printDetails()` yang tidak hanya getter atau setter tetapi memberikan fungsionalitas tambahan untuk mencetak detail person ke konsol. Hal ini memperluas fungsionalitas kelas tanpa harus mengorbankan karakteristik data class yang sederhana.

Before

            // Order.java:
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // setter & getter
}

After

            // Order.java:
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // setter & getter

    // New method added
    public void printDetails() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
    }
}

Dead Code

Ini adalah kode yang tidak lagi digunakan atau diakses oleh bagian lain dari program. Dead code bisa menjadi beban karena memakan ruang dan dapat menyesatkan pemeliharaan karena orang mungkin mengira kode tersebut masih relevan.

Treatment

  • Menghapus code tersebut

Contoh, terdapat method `oldMethod` yang tidak pernah digunakan. Solusinya, kita dapat menghapus method tersebut karena tidak memberikan kontribusi apa pun pada program..

Before

            // Order.java:
public void oldMethod() {
    // Some old logic
}

After

            // Order.java:
// Remove oldMethod

Speculative Generality

Terjadi ketika ada struktur atau fungsi yang dibuat secara berlebihan atau kompleks dengan anggapan bahwa mereka mungkin dibutuhkan di masa depan. Namun, dalam praktiknya, fitur-fitur ini tidak pernah digunakan, yang menyebabkan overhead dan kompleksitas yang tidak perlu.

Treatment

  • Collapse Hierarchy
  • Inline Class
  • Inline Method
  • Remove Parameter
  • Menghapus field tersebut

Contoh, terdapat interface Shape dan implementasi Circle yang tidak digunakan. Solusinya, kita dapat menghapus interface dan implementasi yang tidak terpakai untuk mengurangi kompleksitas dan mempertahankan hanya yang benar-benar diperlukan.

Before

            // Order.java:
interface Shape {
    void draw();
}

// Unused implementation
class Circle implements Shape {
    public void draw() {
        // Draw circle
    }
}

After

            // Order.java:
// Remove unused interface and implementation
